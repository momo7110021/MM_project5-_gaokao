### 3. 五个核心模型概要

1. **分数—位次等效换算**：将跨年度不可直接比较的原始分数统一转换为可比等效分
2. **录取概率预测**：估计考生被目标院校/专业录取的概率及置信区间
3. **专业就业景气度评价**：从就业率、薪资、行业增长、稳定性等多维度评价并输出红黄绿标签
4. **冲稳保志愿组合优化**：生成结构合理（冲/稳/保/垫）的志愿表
5. **志愿填报风险评估**：识别滑档、退档、调剂、冷门、就业、地域六类风险

### 4. 多目标综合效用函数（总模型框架）

max U = alpha * P_admit + beta * M_fit + gamma * E_career + delta * C_city + eta * R_family - lambda * Risk

三种风险偏好：激进型(冲40%,稳30%,保20%,垫10%) / 均衡型(2:3:3:2) / 保守型(1:2:3:4)

## 数据采集方案

设计了7类数据的爬取方案：
1. 一分一段表（省教育考试院 PDF/网页）
2. 院校投档线（省考试院公告）
3. 专业录取数据（高校招生网）
4. 招生计划与限制（招生章程 PDF/网页）
5. 专业就业数据（就业质量报告+国家统计局+招聘网站）
6. 城市产业数据（统计局+统计年鉴）
7. 考生画像（前端问卷录入）

爬取原则：只采集公开数据、不绕过登录/反爬、设置访问间隔、保留来源链接、遵守robots.txt。

## 数据库设计

10张核心表：segment_table(一分一段) / school_info(院校) / major_info(专业) / school_admission_line(投档线) / major_admission(专业录取) / admission_plan(招生计划) / major_employment(就业) / city_data(城市) / candidate_profile(考生画像) / model_output(模型输出)

## 系统接口

统一JSON输出，包含考生信息、等效分、志愿表（逐志愿的概率、匹配度、就业评分、城市评分、风险等级）、风险评估（6类分项风险评分+修改建议）

## 评价体系

14项技术指标（数据爬取成功率>90%、AUC>0.85、Brier<0.15等）
10项业务指标（咨询师使用意愿>70%、家长理解度>80%、人工复核触发率10-30%等）

## 运行验证

代码已通过完整流水线测试，三种方案（aggressive/balanced/conservative）均可正常运行并输出志愿表JSON结果。

## 注意事项

1. 所有数据为模拟数据，仅供教学和测试使用
2. 模型输出概率为基于历史数据的估计值，不构成录取保证
3. 代码中的爬虫URL为示例地址，实际部署需替换为真实地址
4. 论文和代码均遵循"不出现'保证录取'等违规表述"原则
